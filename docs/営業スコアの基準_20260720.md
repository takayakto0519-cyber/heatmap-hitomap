# 営業スコアの基準 — 「営業する価値が高い」をどう数字にしているか

作成：2026-07-20 ／ 対象：運営ダッシュボード「営業（縁の司令室）」タブ

この一枚は、営業リストの並び順（＝どこから当たるか）が**何を根拠に決まっているか**を、
会長がいつでも確認できるようにまとめたもの。画面の「スコアの見方」パネルと同じ内容
（元データは `lib/salesScore.ts` の `SALES_SCORE_CRITERIA`）。

---

## まず結論：スコアは「1個の万能点数」ではなく、意味の違う2種類を並べている

営業リストの並び順は、次の**2種類のスコアを同じ0〜200の物差しに載せて**降順に並べている。
ただし2つは測っているものが違うので、各行に**種別バッジ**を付けて区別している。

| 種別 | バッジ | 何を測るか | 誰に付くか |
|---|---|---|---|
| **縁スコア** | 縁 | 積み上げて“稼いだ”関係の深さ | 縁の台帳のリード（学校・法人） |
| **自治体スコア** | 手動評価 | 手で付けた「提案余地」タグからの見立て | 自治体プロファイル |

> ⚠ 注意：自治体スコアは「提案余地：高」と付けるだけで140点入る**手動評価ベース**。
> 実際に縁を積んだリードより、まだ一度も動いていない自治体が上に来ることがある。
> だから同じ数値でも「縁＝稼いだ数字」「手動評価＝見立て」として読むこと。

これとは別に、各リードには **温度感（🔥/🌤/❄）** と **送信後フォロー状態** が添えられる。

---

## ① 縁スコア（0〜200）＝ 積み上げて“稼いだ”関係の深さ

ヒトマップの方程式そのもの。相手を「落とす」のではなく、足りない要素を埋めるほど上がる。

```
出会い ＝ 事実 × 共感        （どちらかが0なら出会いは0：掛け算）
縁     ＝ 出会い ＋ 行動 × 恩返し
```

| 要素 | 上がり方 |
|---|---|
| 事実（0-10） | 台帳の事実記録・証拠パックの熱いキーワード・連絡先の有無・メモの厚み |
| 共感（0-10） | 「相手の困りごと×自社の強み」を1行書くごとに +4 |
| 行動（0-10） | 一緒に動いた記録ごとに +3（商談中 +2・契約中 +3） |
| 恩返し（0-10） | 先に渡した／お礼・紹介をもらった記録ごとに +4 |

画面に出る数字は「いまの縁の温度」＝ **縁 × 鮮度**（下記②）。

## ② 鮮度（×0.4〜1.0）＝ 何もしないと縁は冷める

最後に手を動かした日からの経過で減衰する。放置への警告。

| 経過 | 係数 | 表示 |
|---|---|---|
| 7日以内 | ×1.0 | 🔥 あたたかい |
| 21日以内 | ×0.85 | 🌤 保たれている |
| 45日以内 | ×0.65 | 🌙 冷めかけ |
| 46日以上 | ×0.4 | ❄ 冷えている |

契約中は関係が続いているため経過日数を半分に見て、冷めにくくする。

## ③ 温度感（🔥熱い/🌤ふつう/❄冷たい）＝ “手がかり”の強さ

縁スコアが「積み上げた深さ」なのに対し、こちらは**証拠パック（メモ）の中身**から見た当たりやすさ。
番人 `lead_temperature` と**同じ基準**（`lib/leadTemperature.ts` の `HOT_WORDS`）。

| きっかけ | 加点 |
|---|---|
| メモに「実証実験」 | +30 |
| メモに「既存」 | +25 |
| メモに「縁」 | +20 |
| メモに「補助金」 | +18 |
| メモに「関係」「公募」「総合計画」 | +12 |
| メモに「指針」「人材不足」「地理的に一致」 | +10 |
| メモに「AI導入」「人材確保」「拠点」 | +8 |
| メモに「DX」「横浜」 | +6 |
| 連絡先（メール/電話）あり | +15 |
| 進行中（接触済み/商談中） | +20 |

**判定：45以上＝🔥熱い ／ 20以上＝🌤ふつう ／ 未満＝❄冷たい**

## ④ 自治体スコア（0〜200）＝ 手動評価ベースの“見立て”

縁スコアのように稼いだ数字ではなく、**手で付けた提案余地タグ＋関わりの段階**から機械換算した目安。

| 提案余地（手入力） | 基礎点 | ＋ 関わりの段階 | 補正 |
|---|---|---|---|
| 高 | 140 | 契約中 | +40 |
| 中 | 80 | 提案済み | +20 |
| 低 | 30 | 様子見 | -10 |

（0〜200にクランプ。ランキングでは「手動評価」バッジで縁スコアと区別）

## ⑤ 送信後フォロー（送信/要フォロー/返信あり）＝ 送った後の鮮度

コールドメールは返信待ちの性質上、縁の鮮度より短いバンドで「そろそろ連絡を」を出す。

| 経過 | 状態 |
|---|---|
| 返信あり | ✓ フォロー不要 |
| 送信〜4日 | 送信N日目（まだ待つ） |
| 5〜9日 | そろそろフォロー |
| 10日以上 | ⏰ 要フォロー |

手動フォロー（電話・対面）を記録すると、その日から数え直す。

## ⑥ ヒトマップ度 fit_score（0〜100）＝ 助成金・コンテストとの相性

助成金/コンテストが「痕跡×地域愛着」というヒトマップの中身にどれだけ合うか。
締切順とは別の「相性順」の並べ替えに使う（70以上＝◎／40-69＝○／未満＝△）。

---

## 使い分けの指針（会長向け）

- **今日どこから当たるか**：営業リスト上位（縁スコア高 or 温度感🔥）から。ただし「手動評価」バッジの自治体は“見立て”なので、動かす前に証拠を一つ確認する。
- **温めるべき相手**：縁スコアはあるのに鮮度が🌙🌙❄になっているもの＝放置で冷えかけ。ひと言便りを。
- **送った後**：⏰要フォローを最優先。返信が無いまま放置＝出会いが冷める。
- **数字を上げたい時**：縁スコアは「共感を1行書く」「一緒に何かする」「先に渡す」で確実に上がる。営業とは点数稼ぎではなく、その行動そのもの。

*基準を変えたら `lib/salesScore.ts` の `SALES_SCORE_CRITERIA` とこの文書の両方を直すこと。*
